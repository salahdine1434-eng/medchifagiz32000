# ميزة «تنظيم الملفات تلقائياً» (AI Medical File Organization)

نظام ذكاء اصطناعي لتحليل وتنظيم الملفات الطبية في مشروع **MedChifaGiz**،
**مدمج داخل لوحة الطبيب** `dr_dashboard.php` كقسم AI جديد، ويعتمد على
**Groq API الموجودة مسبقاً** (الموديل `llama-3.3-70b-versatile`) دون أي اتصال
أو إعدادات جديدة، ودون تغيير أي وظيفة قائمة.

---

## الملفات (جديدة فقط — لم يُعدَّل أي ملف قائم)

| الملف | الدور |
|------|-------|
| `ai_file_organizer.php`      | لوحة القسم (`.ai-content`) التي تُضمَّن داخل الداشبورد |
| `ai_file_organizer_api.php`  | واجهة AJAX/JSON: التحليل + الإحصائيات (تعيد استخدام `db.php` و `GroqReportService`) |
| `ai_file_organizer.js`       | منطق الواجهة + تحليل على دفعات مع شريط تقدّم |
| `ai_file_organizer.css`      | تصميم RTL معزول بالكامل تحت `#afo-root` |
| `ai_file_organization.sql`   | جدول جديد واحد فقط لتخزين نتائج التحليل (Cache) |

> الجدول يُنشأ تلقائياً من الكود عند أول استخدام، فاستيراد ملف `.sql` اختياري.

---

## التركيب

ضع الملفات الخمسة في مجلد المشروع (حيث `dr_dashboard.php` و `db.php` و `groq_report_service.php`)،
ثم أضف **4 أسطر فقط** إلى `dr_dashboard.php` (إضافة محضة — لا حذف ولا تعديل لأي سطر قائم):

**(1) تحميل التنسيق** — بجانب بقية وسوم `<link>` في الأعلى (قرب السطر ~143):
```html
<link rel="stylesheet" href="ai_file_organizer.css">
```

**(2) بطاقة الدخول** — بجانب بقية بطاقات `.ai-card` (قرب السطر ~1180):
```html
<div class="ai-card" onclick="toggleAICard('fileOrganizer')">
    <div class="ai-badge">AI</div>
    <i class="fas fa-folder-tree"></i>
    <h3 data-ar="تنظيم الملفات تلقائياً" data-fr="Organisation des dossiers" data-en="File Organizer">تنظيم الملفات تلقائياً</h3>
</div>
```

**(3) لوحة القسم** — بجانب بقية لوحات `.ai-content` (قرب السطر ~1298، قبل `</section>`):
```php
<?php define('AFO_EMBEDDED', true); include __DIR__ . '/ai_file_organizer.php'; ?>
```

**(4) تحميل السكربت** — بجانب بقية وسوم `<script>` في الأسفل (قرب السطر ~1309):
```html
<script src="ai_file_organizer.js"></script>
```

هذا كل شيء. ستظهر بطاقة «تنظيم الملفات تلقائياً» ضمن بطاقات الذكاء الاصطناعي،
وعند الضغط عليها يُفتح القسم بنفس آلية بقية أقسام AI (`toggleAICard`).

---

## آلية العمل

- يجلب كل ملفات `medical_records` الخاصة بالطبيب الحالي فقط (عزل البيانات).
- يرسل محتوى كل ملف إلى Groq لتحليله، فيُعيد النموذج JSON يحوي:
  التخصص، فئة المرض، الأولوية، الكلمات المفتاحية، الملخص، المتابعة،
  المعلومات الناقصة، والتنظيم المقترح في الأرشيف.
- يخزّن النتيجة في جدول `ai_file_organization` مع **بصمة محتوى (content_hash)**.
- **لا يُعاد إرسال الملف إلى Groq** ما لم يتغيّر محتواه (تُكتشف التغييرات عبر البصمة)
  أو يتغيّر النموذج → عندها يُعاد التحليل ويُحدَّث تلقائياً.
- التحليل يتم **على دفعات** (4 ملفات لكل طلب) لتفادي مهلة PHP، مع شريط تقدّم.
- **تحميل كسول**: لا يُجلب أي شيء حتى يفتح الطبيب القسم أول مرة (لا أثر على سرعة الداشبورد).

لا `if/else` للتصنيف، ولا ترتيب أبجدي، ولا تصنيف يدوي — كل التصنيف من النموذج.

---

## زر «عرض الملف الطبي»

مربوط بنفس آلية أرشيف المرضى في الداشبورد تماماً: يستدعي الدالة العامّة
`openArchiveRecord(medical_record_id)` التي تفتح نفس الـ Modal وتجلب `view_record.php?id=`.
(في الوضع المستقل فقط — للاختبار — يفتح `view_record.php` في تبويب جديد كاحتياطي.)
لم يُعدَّل `view_record.php` ولا منطق الـ Modal إطلاقاً.

---

## الأمان والجودة

- كل الاستعلامات عبر **Prepared Statements** (PDO).
- نفس حارس الجلسة المستعمل في `dr_dashboard.php` (طبيب فقط + دعم موظّف العيادة).
- معالجة دفاعية لاستجابة النموذج (استخراج JSON حتى لو أحاطه نص أو أسوار كود).
- لا كشف لأي تفاصيل حساسة في رسائل الأخطاء (تُسجَّل في error_log فقط).
- لا تعديل على أي جدول قائم — جدول جديد واحد فقط.
- CSS معزول بالكامل تحت `#afo-root` فلا يتعارض مع تصميم الداشبورد.
